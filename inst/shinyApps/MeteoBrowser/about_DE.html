Dateninformationen

Diese Anwendung bietet eine benutzerfreundliche Schnittstelle zum Herunterladen der meteorologischen Daten des Portals Open Data Südtirol.

Die Anwendung Open Data Südtirol API und die dazugehörige Dokumentation sind auf der Webseite der Autonomen Provinz Bozen verfügbar.

Open Data Südtirol - Meteo data web service

Nachdem Sie die Stationen und Sensoren ausgewählt haben, filtern Sie die Tabelle und / oder zeichnen Sie ein Polygon auf der Karte und wählen Sie den gewünschten Zeitraum aus (die verfügbaren Daten beziehen sich auf die letzten 3 Jahre).

Sobald der Daten-Download abgeschlossen ist, wird ein Symbol "speichern" angezeigt und Sie können die Datentabelle im csv- oder json-Format speichern. Sie können zwischen verschiedenen zeitlichen Aggregationen wählen. "grezzo" behält die ursprüngliche Auflösung der Datenerfassung bei. (Die übliche Downloaddauer beträgt 5 oder 10 Minuten). Für die anderen Optionen der zeitlichen Aggregation (Stundenmittelwerte, Tagesmittelwerte, etc.) erhalten Sie andere Statistiken).

Für alle Messgrößen wird die Gesamtzahl der fehlenden Daten für den ausgewählten Zeitraum angezeigt.

Für alle Messgrößen, mit Ausnahme von Niederschlag und Windrichtung, wird der arithmetische Durchschnitt für die spezifische zeitliche Aggregation berechnet.

Niederschlag und Lufttemperatur ergeben sich aus der Gesamtsumme, bezogen auf die spezifische zeitliche Aggregation.

Lufttemperatur und relative Luftfeuchtigkeit beinhalten auch die Minimal- und Maximalwerte, die sich auf die spezifische zeitliche Aggregation beziehen.

Die Windrichtung wird mit den folgenden Zahlencodes neu klassifiziert und der am häufigsten Vorkommende wird den Zahlencodes zugewiesen:

1 = N; 2 = NO; 3 = O; 4 = SO; 5 = S; 6 = SW; 7 = W; 8 = NW.

Sie können zwischen verschiedenen Tabellenformaten wählen, "breit" oder "lange". Das breite Tabellenformat enthält für alle Messgrößen eine entsprechende Spalte. Das lange Tabellenformat beinhaltet eine Spalte, die die Größe "Sensor" identifiziert und eine Spalte "Value" die den Messgrößen entspricht.


Legenda della tabella scaricata

Il significato dei nomi delle colonne o dei codici delle misurazioni è presentato nella seguente tabella:

Bedeutung
TimeStamp Zeitstempel
NAME Stationsname
GS Globalstrahlung
HS Schneehöhe
LD.RED Luftdruck
LF Relative Luftfeuchte
LT Lufttemperatur
N Niederschlag
Q Durchfluss
SD Sonnenscheindauer
W Wassertemperatur
WG Windgeschwindigkeit
WG.BOE Windböe
WR Windrichtung
WT Wassertemperatur

Nel formato di tabella "lungo" sarà presente una colonna "Sensor" con il nome della misurazione (LT,N,WG etc.)

e una colonna "Value" con il valore numerico della misurazione

Se scarichi dati non "grezzi" nel nome delle misurazioni ci sarà una desinenza che indica il tipo di statistica eseguita sull'arco temporale scelto. Di seguito le desinenze indicate:

*_mean : la media dei valori nell'arco temporale

*_sum : la somma dei valori nell'arco temporale

*_max : il valore massimo nell'arco temporale

*_min : il valore minimo nell'arco temporale

*_na : the number of missing values in the timespan

*_Dir : la direzione del vento calssificata come segue:

1 = N; 2 = NE; 3 = E; 4 = SE; 5 = S; 6 = SW; 7 = W; 8 = NW

Haftungsausschluss

Die Qualität der mit dieser Anwendung heruntergeladenen meteorologischen Daten ist nicht garantiert, da sie weder vom Datenanbieter, noch von den Entwicklern dieser WebApp validiert wurden. Diese WebApp hat ausschließlich den Zweck, den Zugriff auf die meteorologischen und Hydrologischen Daten des Portals Open Data Südtirol zu erleichtern.


Starten Sie die Anwendung unter: Von Github heruntergeladen

Der Quellcode ist unter folgendem Link verfügbar:

Giulio Genova - ODBZ

Um die Anwendung zu starten, führen Sie die folgende Zeichenkette aus:


    
if (!require("shiny")) install.packages("shiny")
shiny::runGitHub('GiulioGenova/ODBZ',subdir="app",launch.browser = TRUE)

Dadurch werden automatisch alle notwendigen Pakete auf Ihrem Computer installiert (nur bei der ersten Ausführung erforderlich) und die Webanwendung wird gestartet.
